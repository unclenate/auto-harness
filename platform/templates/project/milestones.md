# Milestones

<!-- Source: platform/profiles/management/project-standard -->
<!-- Update status at each governance cadence or when a milestone completes / slips. -->

Milestones mark the completion of a meaningful phase of work. A milestone is not a task —
it is a point at which the project can be assessed, demonstrated, or handed off. Each
milestone should have explicit exit criteria so "done" is unambiguous.

**Status definitions:**

- **Planned** — Scheduled; work has not started
- **Active** — Work in progress; currently on track
- **Done** — All exit criteria met and verified
- **Slipped** — Target date passed; add a revised target and reason in the Notes column
